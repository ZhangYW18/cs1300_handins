Analysis for Each Service

Spotify

Most successful point: Spotify implemented rich features that other streaming service does not have. The playlists can be sorted in multiple orders and viewed in multiple visual modes, and it's very easy for users to utilize - just click the button in the top right corner of library, then select the action, and it's done. It's also possible to add a song to multiple playlists at one time. Those features greatly increases the usability of the website.

Least successful point: Playlists cannot be directly shared on social media like Facebook and Twitter, which makes it harder for users to share their taste in music.

Apple Music

Most successful point: Use of icons is everywhere for every feature that is implemented for playlist, which is good for learnability and usability. For example, in the menu above for a playlist, there's one icon for almost each choice in the menu, making it easier for users to learn what each choice is about and distinguish between similar choices (Play Next vs Play Later).

Least successful point: The least successful point is related to filter and sorting. Playlists cannot be filtered or sorted by users, showing low operability of the website. Playlists can only be sorted in default order by name, but result is wrong when names containing multiple languages. For example, in the above screenshot, last playlist is named "Tamayura" in Japanese but sorted to the last. This shows its disadvantages in terms of accessibility for multiple languages.

Amazon Music

Most successful point: They are the only website out of all 3 websites that can recommend songs based on your playlist. This allows more detailed personalized recommendation based on different playlist instead of based on songs listened by a user generally.

Least successful point: The website is visually monotonous and lacks attractiveness. All playlists are laid out in grids, with their cover being one or some of the covers of the songs in playlist. With few playlists created, the right part of the screen is completely empty below the header.

Inspiration for My Project

Based on the analysis above, I have a list of features that I want to implement for my app:

  • My app is an album favorite page, where all albums are listed, and users can choose which to favorite.
  • Each album has a cover image, a title, an artist, a genre and a release year.
  • Albums can be filtered by genre or time.
  • Inspired by Apple Music's shortcoming, I decide to include sorting feature for albums in my app. Sorting key can be the title, artist or release year of the albums.
  • Inspired by Amazon Music's shortcoming, I decide to let the albums be spaced evenly in one row when there are only one row displaying albums in total.

Implementation

Overview

The intention of my app is to serve as an album library page in a music streaming service, so that users could choose albums to favorite or remove albums from favorites. Also, users could filter ot sort albums in the library.

Link to Completed Work: https://cs1300-ass4.vercel.app/

The albums are shown in the left part of the page under the title "My Albums". Other features like filtering, sorting, aggregator are put in the right part of the page. Depending on the genres of albums, we'll collect genre tags for users to filter. Users can also filter albums by set the release time range. We provide three sorting keys for albums: title, artist, year.

For sorting, we firstly sort them by the key user choose. Then to solve ties, we use other keys in the following order: artist, title, year, after removing the key user choose from this list.

When there's only one row displaying albums, we place all albums evenly in that row like the above screenshot.

Interesting Lessons Learned

From this project, I learned that many companies prioritize their app versions over their web counterparts, and provide richer features on apps. Possibly the intention is to encourage users to download their app rather than using the web version. This could also be due to the relatively small user base on the web version.
